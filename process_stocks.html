Manual prediction using ARIMA¶

Before making prediction, it is necessary to analyze the stationarity of the timeseries. This is necessary because most prediction models (including the one used in this task - ARIMA) assume the stationarity of the time series. For a stationry time series, the models work on the assumption that the behaviour will repeat in the future

It is not very clear visually whether the time series of this stock is stationary, although it looks like there is no trend (stays near 0).

In addition, we need to have a way to automatically test the stationarity of all the stocks in a more automatic way. This needs to be done before we can apply the prediction model for each stock. To this end, Augmented Dickey-Fuller Test can be used.

More information can be found here: https://en.wikipedia.org/wiki/Augmented_Dickey%E2%80%93Fuller_test

The null hypothesis of the Augmented Dickey-Fuller (ADF) is that there is a unit root (non stationary), with the alternative that there is no unit root (the time series is stationary). If the pvalue is above a confidence level, then the time series is non-stationary.

Below is a function that takes the series, performs the ADF test and output whether the time series of each stock is stationry.

With the confidence level of $1%$, the test indicates that only 3 (out of 1000 stocks) are non stationary. It is safe to assume that the movements of close_rtn of most of the stocks are stationary.

We can proceed building the model.

In [139]:
from statsmodels.tsa.stattools import adfuller
def is_stationarity(series,conf_level=5):
    series=series.dropna()
    try:
        tests = adfuller(series)

        #compare the test statistics vs the according critical value
        test_statistic=tests[0]
        critical_val=tests[4]['{}%'.format(conf_level)]

        return True if test_statistic<critical_val else False
    except ValueError:
        return None
    
    return False

PREDICTION Using ARIMA.¶

More details of ARIMA can be found here

Once we confirm the time series are stationary, an important task is to choose the parameters p, d, q for the algorithm.

  1. p is the number of lags, i.e., how far back the model will use the values to predict the current value. For example, if p is 3, the model will look back and use the previous 3 values.
  2. d
  3. q is the number of lagged moving averages. For example, if q is $3$, the model will incorporate the last $3$ errors (difference between the moving average and the actual value)

Setting the values of p, d, q can be a problem because the meaning of each parameter is not very intuitive. Let's use a grid search to find the best parameters.
